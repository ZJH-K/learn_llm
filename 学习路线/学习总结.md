# 阶段一 · Python 高级语法

## Week 1 · 语法核心

### Day 1 · 装饰器基础(2026-04-18)

🎯 装饰器 = 吃函数吐函数的高阶函数。`@timing` ≡ `slow = timing(slow)`

📌 三层骨架

| 层 | 写法 | 干什么 |
|---|---|---|
| 外 | `def timing(func)` | 接被装饰函数 |
| 中 | `def wrapper(*a, **kw)` | 透传参数的替身 |
| 内 | `result = func(*a, **kw)` | 调一次原函数 |

💡 核心规则
1. wrapper 必须 `return result`,否则原返回值被吃掉变 `None`
2. `@functools.wraps(func)` 保 `__name__` / `__doc__`
3. `*args, **kw` 是签名透传万能模板,所有装饰器都这么写
4. 计时一律 `time.perf_counter()`(单调时钟),不用 `time.time()`

⚠️ 坑
- wrapper 忘 return → 拿到 None
- `wraps` 拼成 `wrap` → AttributeError
- `time.time()` 受系统时间校正漂移干扰

📝 最小代码
```python
import functools, time

def timing(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        t0 = time.perf_counter()
        result = func(*args, **kwargs)
        print(f"{func.__name__}: {(time.perf_counter()-t0)*1000:.2f}ms")
        return result
    return wrapper
```

🔑 **外吞内替、参数透传、return 别忘、wraps 保身**

### Day 2 · 装饰器进阶(2026-04-19)

🎯 带参装饰器 = 三层嵌套(吃参数→吃函数→执行);类装饰器 = `__init__` 存状态 + `__call__` 执行

📌 几种形态对比

| 形态 | 等价展开 | 骨架嵌套 |
|---|---|---|
| `@timing` | `f = timing(f)` | 2 层 |
| `@retry(times=3)` | `f = retry(3)(f)` | 3 层 |
| `@Counter` | `f = Counter(f)` | 类:`__init__` 接 func |
| `@Retry(times=3)` | `f = Retry(3)(f)` | 类:`__init__` 接配置, `__call__` 接 func |

💡 核心规则
1. **带参装饰器必须写括号**,哪怕全默认也要 `@retry()`
2. 类装饰器**模块加载时只装一次**,实例固定 → `count` 等状态会一路涨
3. 装饰器叠加 `@A @B def f` ≡ `f = A(B(f))`,**装从内到外,调从外到内进出洋葱**
4. `try/except` 不是 `if/else`:**try 默认走通,抛异常才跳 except**
5. 闭包**修改**外层变量必须 `nonlocal`,否则被当局部变量 → `UnboundLocalError`(可改用 `dict["n"]` 绕)

⚠️ 坑
- `@retry` 漏括号 → `TypeError: missing 1 required positional argument`(Python 不查语义类型,只查参数个数)
- `except Exception:` 把 bug 型异常也吞了,导致重试掩盖真错
- 闭包 `counter += 1` 不加 nonlocal → UnboundLocalError

📝 最小代码
```python
def retry(times, delay):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kw):
            for i in range(times):
                try: return func(*args, **kw)
                except Exception:
                    if i == times - 1: raise
                    time.sleep(delay)
        return wrapper
    return decorator
```

🔑 **多一层括号多一层嵌套;闭包改值必声明;洋葱叠装从内到外、调用从外到内**

💎 加分
- 类装饰器适合"需要暴露 `.reset()` / `.count` 接口"的场景,函数装饰器写不出
- `@A @B` 想最先看日志的写最上面
- `return` 一执行函数立即结束,后面是 unreachable code

### Day 3 · 生成器 + yield(2026-04-19)

🎯 generator = **可暂停可恢复的函数**。函数体里出现 `yield` 就自动变 generator,调用不执行只返回对象,`next()` 才推进到下个 yield 冻结

📌 三大模式

| 模式 | 写法 | 用途 |
|---|---|---|
| 单流 | `def g(): yield x` | 流式吐数据 |
| 管道 | `g2(g1(...))` | 数据"流"过不落地 |
| 透传 | `yield from sub()` | 整批转发 |

💡 核心规则
1. **函数里有 yield = generator**,没有 yield = 普通函数(只写 print 不写 yield → 不会吐任何东西)
2. **yield 是暂停键,不是终止键** → `yield x; do_more()` 的 `do_more` 要下次 next 才跑
3. **generator 只能遍历一次**,用完即废,要重来必须重新调函数拿新对象
4. `yield from` 适合"整批透传",过滤/变换场景用 `for+if+yield`,**别乱用**
5. **不能 `len(generator)`**,数个数用 `sum(1 for _ in g)`

⚠️ 坑
- `yield line; line.rstrip()` → rstrip 对已吐出的值无效(yield 已暂停)
- `line.rstrip("\n")` 单独一行不接返回值 → str 不可变,白跑(写 `yield line.rstrip("\n")`)
- `for x in 3` 报错 → int 不可迭代

📝 最小代码
```python
def read_lines(path):
    with open(path) as f:
        for line in f:
            yield line.rstrip("\n")

def filter_keyword(lines, kw):
    for line in lines:
        if kw in line:
            yield line

# 管道:数据流不落地
for hit in filter_keyword(read_lines("a.log"), "ERROR"):
    print(hit)
```

🔑 **yield 是暂停键、yield from 是转发键、generator 是管道里一节水管**

💎 加分
- `for x in g` ≡ `it = iter(g); while True: try x=next(it) except StopIteration: break`
- `yield from sub()` 能接收 sub 的 `return` 值(在 `StopIteration.value` 里),协程双向通信前身

### Day 4 · 迭代器协议(2026-04-19)

🎯 `__iter__` + `__next__` + `StopIteration` = 可被 `for` 遍历的合同

📌 Iterable vs Iterator 核心区分

| | Iterable(数据本体) | Iterator(一次性游标) |
|---|---|---|
| 实现 | `__iter__` | `__iter__` + `__next__` |
| 能 `for` | ✅ | ✅ |
| 能 `next()` | ❌ | ✅ |
| 多次遍历 | ✅(每次新游标) | ❌ 用完即废 |
| 例子 | list / tuple / dict / str | generator / iter(lst) / 文件对象 |

💡 核心规则
1. **`iter(x) is x` 为 True ⟺ x 是 Iterator** —— 自己写 Iterator 时 `__iter__` 必须 `return self`
2. **class Iterator 的 `__next__` 不写循环**(Python 自动反复调)
3. **generator 必须写 `while True`**(否则跑到底就 StopIteration)—— class 不写循环、generator 必写循环,**位置正好相反**
4. generator 是 Iterator 协议的语法糖,Python 自动帮你实现协议

⚠️ 坑
- `next([1,2,3])` 炸 → list 是 Iterable 不是 Iterator,要先 `iter(...)`
- class `__next__` 里写 `while True` → 死循环
- 一个 Iterator 跑两遍 for → 第二遍空,要重新拿对象

📝 最小代码
```python
class Fib:
    def __init__(self):
        self.a, self.b = 0, 1
    def __iter__(self):
        return self                               # ⚠ Iterator 必须返回 self
    def __next__(self):
        v = self.a
        self.a, self.b = self.b, self.a + self.b
        return v
```

🔑 **Iterable 是书,Iterator 是书签;class 不写循环,generator 必写循环**

💎 加分
- 简单吐序列用 generator;需要 `.reset()` 接口用 class Iterator;想多次遍历用 class Iterable(`__iter__` 返回新 Iterator,像 list)

### Day 5 · 上下文管理器(2026-04-19)

🎯 `with` = **try/finally 的语法糖**,专门解决"拿了必须还"的资源场景(文件 / 连接 / 锁)

📌 两种实现对比

| 维度 | class 版 | `@contextmanager` 版 |
|---|---|---|
| `as xxx` 赋什么 | `__enter__` 返回值 | yield 后的表达式 |
| 感知异常 | `__exit__` 的 `exc_type` 参数 | `try/except` 在 yield 处捕获 |
| 异常向上抛 | 默认 `return None` 自动抛 | `except` 必须显式 `raise` |
| 清理代码 | `__exit__` 方法体 | `finally` 块 |

💡 核心规则
1. `__exit__` 默认 `return None` → 异常向上抛(我们要的);**`return True` 会吞异常**(慎用)
2. `@contextmanager` 的 yield **必须包 try/finally**,否则异常路径下清理全漏
3. `except` 里**忘 `raise` → 异常被静默吞**(看起来正常实际已炸)
4. 选型:简单进出用 `@contextmanager`(代码省一半);要细分异常类型 / 需要跨方法共享状态用 class

⚠️ 坑
- dunder 拼错 Python 不报错(`__intit__` 变普通方法,实例化时 `self.x` 没设)
- 裸 `except:` 吞 Ctrl-C(用 `except Exception:`)
- f-string `{}` 里只放变量,其他都是字面字符(不要写 `f"{exec: <sql>}"`)

📝 最小代码
```python
from contextlib import contextmanager

@contextmanager
def db_session():
    conn = connect()
    try:
        yield conn
        conn.commit()
    except Exception:
        conn.rollback()
        raise                # ⚠ 必须显式 raise
    finally:
        conn.close()
```

🔑 **with = 自动 try/finally;class 默认透传异常,contextmanager 要显式 raise**

### Day 6 · 魔法方法 + 动态属性(2026-04-20)

🎯 dunder 方法 = Python 语法糖的接插件,让自定义类融入 `print` / `==` / `()` / `.` 原生语法

📌 5 个核心方法

| 方法 | 触发时机 | 干什么 |
|---|---|---|
| `__init__` | `Foo(...)` | 实例化 |
| `__call__` | `obj(...)` | 实例当函数调 |
| `__repr__` | `print(obj)` / f-string / list 显示 | 调试字符串 |
| `__eq__` | `obj == x` | 自定义相等 |
| `__getattr__` | `obj.x` **正常查找失败**的兜底 | 动态属性 / 点取 dict |

💡 核心规则
1. `__getattr__` 是查找**失败**的兜底,不是每次访问都触发(那是 `__getattribute__`)
2. `__eq__` 类型不匹配 **`return NotImplemented`** 不是 `return False`(让对方 `__eq__` 试,保对称性)
3. **重写 `__eq__` 后 `__hash__` 自动变 None** → 对象不能进 set / dict key,要支持就同时重写 `__hash__`(用同一组字段)
4. EAFP 优于 LBYL:`try: d[k] except KeyError` 比 `if k in d: d[k]` 命中只查 1 次
5. `f"{v!r}"` 调 `repr(v)`(字符串带引号),嵌套对象自动递归展开

⚠️ 坑
- `__getattr__` 里访问 `self._data`,若 `_data` 未初始化会**无限递归**(防御:第一行守 `if k == "_data": raise AttributeError(k)`)
- `if v:` 把 0 / False / "" / [] 误判为"不存在",要用 `if k in d`
- `.values` 漏括号变方法对象,永远不等

📝 最小代码
```python
class Config:
    def __init__(self, data):
        self._data = {k: (Config(v) if isinstance(v, dict) else v) for k, v in data.items()}
    def __getattr__(self, k):
        try: return self._data[k]
        except KeyError:
            raise AttributeError(f"{type(self).__name__!r} has no attr {k!r}") from None
    def __repr__(self):
        return f"Config({', '.join(f'{k}={v!r}' for k, v in self._data.items())})"
    def __eq__(self, other):
        if not isinstance(other, Config): return NotImplemented
        return self._data == other._data
```

🔑 **getattr 是查找兜底、eq 不匹配返 NotImplemented、重写 eq 别忘 hash**

💎 现实对应:Pydantic `BaseModel` / OmegaConf / HuggingFace `AutoConfig` / `argparse.Namespace` 底层都是这套

### Day 7 · Week 1 综合复盘(2026-04-21)

🎯 综合作业 `tinylog` = 装饰器 + 上下文管理器 + 生成器三件套同时激活,验证 Week 1 单点能否拼成整体

📌 三件套各管什么

| 机制 | 角色 | 在 tinylog 里 |
|---|---|---|
| 装饰器 | 函数包装 | `@log_calls` 装饰处理函数 |
| 上下文管理器 | 带清理的代码块 | `with log_block("批次")` |
| 生成器 | 流式吐数据 | `tail(path)` 行流 |

💡 核心规则
1. 装饰器 `start = time.perf_counter()` **必须放 wrapper 内**,放外层只算装饰那一刻
2. **`finally: return` 会吞异常** —— 异常路径用 `except: ...; raise`,不用 finally
3. **func 在装饰器里只能调一次**,`result = func(...)` 存起来 → 副作用函数(发邮件 / 转账)双调 = 生产事故
4. **重写 `__eq__` 自动让 `__hash__ = None`** → 对象不能进 set,要支持要同时写 `__hash__`(用同一组字段)
5. **类型注解运行时不做检查**,只服务 IDE / mypy / Pydantic / FastAPI 这种主动读注解的库

⚠️ 坑
- `func.__name__(*args)` 把字符串当函数调 → TypeError(`__name__` 是属性不是方法)
- `except Exception: raise` 是无用模式 = 没抓
- `tail` 把 `with open` 放生成器内 → 文件什么时候关取决于生成器**耗尽 / GC / .close()**,不是 caller for 结束

📝 最小代码
```python
def log_calls(func):
    @functools.wraps(func)
    def wrapper(*args, **kw):
        start = time.perf_counter()                # ⚠ 在 wrapper 里
        try:
            result = func(*args, **kw)             # ⚠ 只调一次
        except Exception as e:
            elapsed = time.perf_counter() - start
            print(f"!!! {func.__name__} {type(e).__name__}: {e} after {elapsed:.4f}s")
            raise                                  # ⚠ 重抛
        elapsed = time.perf_counter() - start
        print(f"<<< {func.__name__} → {result!r} in {elapsed:.4f}s")
        return result
    return wrapper
```

🔑 **start 在 wrapper 内、func 只调一次、finally 别 return、异常该抛就抛**

💎 类型注解
- `list[int]` / `dict[str, int]`(3.9+ 内建)优先于 `typing.List`
- `int | str`(3.10+)替代 `Union[int, str]`,`str | None` 替代 `Optional[str]`

## Week 2 · 异步 + 工程化

### Day 8 · asyncio 基础(2026-04-21)

🎯 单线程并发:协程等 I/O 时让出 CPU 给别人,**异步治"等待"不治"算得慢"**

📌 四件套

| 关键字 / API | 作用 |
|---|---|
| `async def foo()` | 定义协程函数。**调用不执行**,只返回协程对象 |
| `await x` | 等 x 完成,期间让出 CPU。只能在 async def 里用 |
| `asyncio.run(协程)` | 启动事件循环,程序通常只调一次 |
| `asyncio.gather(*coros)` | 并发跑多个,等全部完成,**按传入顺序**返回结果 |

💡 核心规则
1. **判用不用异步看任务**:I/O 密集(爬虫 / LLM / DB)✅,CPU 密集(矩阵 / 加密)❌ 该上多进程
2. **顺序 `await` 多个协程 = 串行**,光写 async 不加速,真正并发靠 `gather` / `create_task`
3. **`gather(coros)` ≠ `gather(*coros)`** —— gather 要的是多个位置参数,不是一个容器
4. 异步代码里**不能混同步 I/O**(`httpx.get` 阻塞整个事件循环)
5. `httpx.AsyncClient` 是类要实例化:`async with httpx.AsyncClient() as c`,不是 `async with httpx.AsyncClient as c`

⚠️ 坑
- `async def foo()` 调用不 await → `coroutine 'foo' was never awaited` 警告(头号坑)
- 每个请求开新 client → 重建连接池,十请求十套握手(共享 client 提速 41%)
- `g([1,2,3])`(`def g(*args)`)→ args 是 `([1,2,3],)`,不是 `(1,2,3)`(调用处 `*` 才是拆包)

📝 最小代码
```python
import asyncio, httpx

async def fetch(client, url):
    r = await client.get(url)
    return r.status_code

async def main():
    urls = ["https://example.com"] * 10
    async with httpx.AsyncClient() as client:        # 共享 client
        results = await asyncio.gather(*(fetch(client, u) for u in urls))
    print(results)

asyncio.run(main())
```

🔑 **async 不加速 await 才让出、gather 必须解包、共享 client 别重建**

💎 `*` / `**` 速记
- 定义时打包:`def f(*args)` → tuple,`def f(**kw)` → dict
- 调用时拆包:`f(*lst)` 等价 `f(1,2,3)`,`f(**d)` 等价 `f(a=1, b=2)`
- 一句话:**定义时合,调用时拆**

### Day 9 · async 深入:Task / Semaphore / wait_for / cancel / Queue(2026-04-21)

🎯 `gather` 是发动机,Day 9 学的是控制发动机的五个旋钮

📌 五旋钮

| 旋钮 | 作用 | 何时用 |
|---|---|---|
| `create_task` | 起步发车,可先发后等 | fire-and-forget(心跳 / LLM 流式) |
| `Semaphore(N)` | 同时最多 N 个 | 控并发数(防 429 / 封 IP) |
| `wait_for(coro, t)` | 超时自动 cancel | 防下游卡死拖垮自己 |
| `task.cancel()` | 注入 CancelledError | 用户中断 / 主程序关后台 |
| `Queue(maxsize)` | 生产消费解耦 + 背压 | 速度不匹配场景 |

💡 核心规则
1. **Semaphore 甜点公式**:`并发 ≈ rate × 平均耗时 × 0.5`(留 buffer)
   - OpenAI 500rpm × 3s ≈ 10-20;本地 1000 文件 ≈ 50-100;MySQL 池 20 ≈ 15
2. **超时异常类型谁管谁抓**:`httpx.AsyncClient(timeout=3)` → `httpx.TimeoutException`;`asyncio.wait_for` → `asyncio.TimeoutError`,**抓错 = 没抓**
3. **取消三铁律**:cancel 后必须 await task / 抓 CancelledError 必须 raise / 用 `except Exception` 而不是裸 except 防误吞(3.8+ 已让 CancelledError 继承 BaseException)
4. **Queue 速度真理**:加生产者不提速,**加消费者才提速**;系统稳态吞吐 = 消费者数 / 单消费耗时
5. `await t1` 期间所有 task 都在跑,**完成顺序只看各自耗时**,跟 await 谁无关

⚠️ 坑
- 后台 task 异常**静默丢失**:consumer 崩了 main 不知道,`queue.join()` 死等 → 死锁(防护:每个 task 外包 try/except 暴露错误)
- `time.sleep` 用在异步里 → 同步阻塞整个事件循环;`await time.sleep` → TypeError(`time` 只留给 `perf_counter`)
- `task_done` 拼成 `task_down` —— done 是完成,不是下去

📝 最小代码
```python
async def fetch(client, sem, url):
    async with sem:                                   # 令牌:用了自动还
        try:
            r = await asyncio.wait_for(client.get(url), timeout=3.0)
            return r.status_code
        except (asyncio.TimeoutError, httpx.TimeoutException):
            return -1                                  # 抓两种类型

async def main():
    sem = asyncio.Semaphore(3)
    async with httpx.AsyncClient() as client:
        results = await asyncio.gather(*(fetch(client, sem, u) for u in urls))
```

🔑 **Semaphore 防炸下游、wait_for 防被下游炸、cancel 三铁律不能破、Queue 加消费者才提速**

💎 工程词汇
- **下游 / 上游**:谁调我 = 上游(用户 / 前端),我调谁 = 下游(LLM / DB)。"下游超时 / 限流 / 抖动"
- **线程池**:预创建 worker 线程池,asyncio 默认 = `min(32, CPU+4)`;`aiofiles` 底层就是它
- **HTTP 429 / 503**:OnCall 最常见两个告警;429 = 限流(客户端锅),503 = 过载(服务端锅)
- **解耦**:改 A 不被迫改 B;Queue 是最典型解耦工具
