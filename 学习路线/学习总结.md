## 阶段一:Python 高级语法

### Week 1 · 语法核心

#### 装饰器(Day 1 · 2026-04-18)

**是什么**
装饰器本质就是一个**高阶函数**:吃一个函数进来,吐一个"包装过的新函数"出去。`@timing` 写在 `def slow` 上面,只是语法糖,真正发生的事是:

```python
slow = timing(slow)   # @timing 等价于这一行
```

**为什么需要**
想给一堆已有函数"叠一层能力"(计时、重试、日志、鉴权),又不想去改每个原函数的代码。装饰器就是 Python 里"横切关注点"的标准解法 —— 功能代码和业务代码分开。

**核心用法(三层骨架)**

```python
import functools, time

def timing(func):                         # 第一层:接收被装饰的函数
    @functools.wraps(func)                # 保住原函数的 __name__ / __doc__
    def wrapper(*args, **kwargs):         # 第二层:真正替身,透传参数
        t0 = time.perf_counter()
        result = func(*args, **kwargs)    # 调一次原函数
        print(f"{func.__name__} 耗时 {(time.perf_counter()-t0)*1000:.2f} ms")
        return result                     # ⚠ 必须把结果 return 出去
    return wrapper                        # 第一层返回 wrapper 替换掉原函数

@timing
def add(a, b): return a + b
```

**典型坑(自己踩过的 3 个)**
1. **wrapper 忘 return 原函数结果** → 调用 `add(2,3)` 拿到的是 `None`,因为 wrapper 默认返回 `None`,原函数的返回值被吃掉了
2. **`functools.wraps` 拼成 `wrap`**(单数) → `AttributeError`,因为 `wrap` 是另一个不相关的东西
3. **计时用 `time.time()`** → 它是墙钟,会被系统时间校正跳动干扰。想测"代码跑了多久"一律用 `time.perf_counter()`,这是单调时钟,只增不减

**附:万能参数 `*args, **kwargs`**

- `*args`:把"位置参数"打包成元组。`def f(*args)`,调用 `f(1, 2, 3)` → `args = (1, 2, 3)`
- `**kwargs`:把"关键字参数"打包成字典。`def f(**kwargs)`,调用 `f(a=1, b=2)` → `kwargs = {"a":1, "b":2}`
- 在装饰器里必须写 `def wrapper(*args, **kwargs)` + `func(*args, **kwargs)` → **签名透传**:无论原函数长什么样(要几个参数、要不要关键字参数),wrapper 都能原样接住再原样喂回去
- `*` 和 `**` 出现在**函数定义**里是"打包",出现在**函数调用**里是"拆包":`f(*args)` 等价于把元组里的元素一个一个展开当位置参数传进去
- 记忆锚:**定义时打包成集合,调用时拆包发出去**

**一句话记忆**
装饰器 = "外层函数吞函数、内层 wrapper 替身干活、`*args/**kwargs` 透传参数、别忘了 return、元信息靠 wraps 保住"。
